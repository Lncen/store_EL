# 提交到git

```sh
git commit --no-verify -m "这里写说明"
```

### 提交规范类型

- 例子:
  - feat: 新增用户表

📌 可用的常见提交类型（type）| 类型 | 含义说明 | |---------|------------------------------| | feat | 新功能 | | fix | 修复 bug | | chore | 构建/工具/依赖更新等 | | docs | 文档更新 | | style | 样式改动（不影响逻辑） | | refactor| 重构代码 | | perf | 性能优化 | | test | 测试相关 | | build | 构建系统或包依赖变更 | | ci | CI 配置修改 | | revert | 回滚提交 |
